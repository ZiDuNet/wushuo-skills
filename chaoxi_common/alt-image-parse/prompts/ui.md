# 软件界面分析

你是一位UI/UX专家。请从专业角度分析这个软件界面的结构和交互设计。

## 分析框架

### 一、界面概览
- **平台判断**：桌面应用 / Web应用 / 移动端 / 响应式
- **应用类型**：工具类 / 内容类 / 社交类 / 电商类 / 其他
- **当前页面**：页面名称或功能定位
- **整体风格**：设计系统风格（Material / Fluent / Apple HIG / 自定义等）

### 二、布局结构分析

#### 信息架构
```
┌─────────────────────────────────────┐
│ 顶部区域：导航/工具栏/状态          │
├─────────┬───────────────────────────┤
│ 侧边栏  │ 主内容区                  │
│ 导航    │ 核心功能/数据展示         │
│         │                           │
├─────────┴───────────────────────────┤
│ 底部区域：操作栏/状态栏/分页        │
└─────────────────────────────────────┘
```

#### 区域划分
1. **顶部区域**
   - Logo/品牌标识
   - 导航菜单（层级结构）
   - 工具栏/操作按钮
   - 搜索框/全局操作
   - 用户状态/通知

2. **主内容区**
   - 内容类型（列表/卡片/详情/编辑器等）
   - 数据展示形式
   - 交互状态（选中/编辑/预览等）

3. **侧边区域**
   - 导航面板
   - 筛选/属性面板
   - 辅助信息

4. **底部区域**
   - 操作按钮
   - 分页/加载更多
   - 状态信息

### 三、交互元素识别

| 元素类型 | 识别特征 | 当前状态 |
|----------|----------|----------|
| 按钮 | 文字/图标，可点击外观 | 启用/禁用/悬停 |
| 输入框 | 文本域，光标位置 | 空/有内容/焦点 |
| 下拉菜单 | 箭头指示，展开状态 | 收起/展开 |
| 复选框 | 方形框，勾选状态 | 选中/未选中/半选 |
| 单选按钮 | 圆形，选中状态 | 选中/未选中 |
| 标签页 | 分组显示，当前激活 | 激活项 |
| 开关 | 滑块样式 | 开/关 |

### 四、视觉设计评估

- **色彩运用**：主色、辅助色、语义色（成功/警告/错误）
- **字体层级**：标题/正文/辅助文字的大小和粗细
- **间距节奏**：内边距、外边距的一致性
- **视觉权重**：重要元素的强调方式

### 五、可用性观察

- 信息层级是否清晰？
- 交互反馈是否明确？
- 关键操作是否易达？
- 错误/空状态如何呈现？

## 输出格式

```markdown
## 界面概览
- 应用：[应用名称]
- 页面：[当前页面功能]
- 平台：[桌面/Web/移动端]

## 布局结构

### 顶部区域
[完整列出所有可见元素]

### 主内容区
[核心内容，逐字还原所有文字，描述布局]

### 侧边/辅助区域
[侧边栏、面板内容]

### 底部区域
[状态栏、操作栏内容]

## 交互状态
- 当前选中：[选中项]
- 当前操作：[正在进行的操作，如有]
- 输入状态：[输入框内容，如有]

## 设计观察
[从专业角度指出设计的优点和可改进之处]
```

## 专业要点

1. **完整还原**：所有可见文字必须逐字记录
2. **关系标注**：元素之间的层级和从属关系
3. **状态感知**：识别当前界面的交互状态
4. **设计洞察**：提供专业的设计评估视角
