# AI 图片解析

根据图片和分析类型，进行内容分析、结构化数据提取、OCR 文字识别或提示词逆向。

---

## ⚠️ 重要：必须选择正确的 --type

**`--type` 参数不是可选的装饰，它决定了分析的专业性和深度。**

| 场景 | 错误做法 | 正确做法 | 效果差异 |
|------|---------|---------|---------|
| 软件截图 | `--type default` | `--type ui` | UI专家视角 vs 通用描述 |
| 摄影作品 | `--type default` | `--type photo` | 构图/用光分析 vs 基础描述 |
| 数据图表 | `--type default` | `--type chart` | 精确数据提取 vs 大概描述 |
| 广告海报 | `--type default` | `--type marketing` | 传播效果评估 vs 内容罗列 |

**每个 type 都有专门设计的专业分析框架，使用错误的 type 会浪费这些能力。**

---

## 类型选择决策逻辑

**选择优先级：用户意图 > 图片类型判断 > default**

### 第一优先级：用户明确意图

根据用户描述中的关键词直接匹配：

| 用户说... | 关键词 | 选择 |
|----------|--------|------|
| "识别文字"、"读出内容"、"OCR" | 识别/提取/读出/文字 | `--type ocr` |
| "提取表格"、"表格数据" | 表格/数据表 | `--type table` |
| "分析界面"、"这个APP"、"网页截图" | 界面/APP/网页/软件 | `--type ui` |
| "配色怎么样"、"设计分析" | 设计/配色/排版 | `--type design` |
| "构图怎么样"、"用光分析" | 照片/构图/用光/摄影 | `--type photo` |
| "提取图表数据"、"柱状图" | 图表/柱状图/折线图/饼图 | `--type chart` |
| "怎么生成的"、"反推提示词" | 逆向/反推/AI生成 | `--type reverse` |
| "聊天记录"、"微信截图" | 聊天/对话/微信/QQ | `--type chat` |
| "这个广告怎么样"、"宣传效果" | 广告/宣传/营销/海报文案 | `--type marketing` |

### 第二优先级：图片类型自主判断

**用户没有明确意图，但你能从图片来源/上下文判断类型时，主动选择：**

| 图片是什么 | 选择 | 依据 |
|-----------|------|------|
| 软件截图、APP界面、网页 | `--type ui` | 有界面元素、按钮、菜单 |
| 数据表格、Excel截图、报表 | `--type table` | 有行列结构、数据单元格 |
| 统计图表、柱状图、折线图 | `--type chart` | 有坐标轴、数据可视化 |
| 摄影作品、风景人像 | `--type photo` | 真实场景拍摄 |
| 海报、广告、宣传物料 | `--type marketing` 或 `--type design` | 有文案+视觉设计 |
| 聊天截图、对话记录 | `--type chat` | 有气泡、头像、时间戳 |
| AI生成图片、插画 | `--type reverse` | 有明显AI特征 |
| 文档扫描、合同票据 | `--type ocr` | 主要是文字内容 |

### 第三优先级：default 兜底

仅在以下情况使用 `--type default`：
- 用户无明确意图 **且** 无法判断图片类型
- 图片内容混合、不属于任何明确分类
- 用户要求"描述一下"、"看看这是什么"等通用需求

---

### 决策流程图

```
用户是否有明确意图？
│
├─ 是 → 关键词匹配 → 选择对应 type
│
└─ 否 → 能否从上下文判断图片类型？
        │
        ├─ 能 → 选择对应 type
        │
        └─ 否 → --type default
```

## 各类型详解

### `ocr` - 文字识别
**核心能力**：逐字精确、结构还原、多栏处理
**适用**：文档扫描、截图文字提取、合同票据
**输出**：完整文字内容，保持原有层级和格式

### `table` - 表格提取
**核心能力**：复杂表格解析、合并单元格处理、数据类型识别
**适用**：Excel截图、数据表、报表
**输出**：Markdown表格，数值保持精度

### `ui` - 界面分析
**核心能力**：信息架构、交互元素、可用性评估
**适用**：APP截图、网页截图、软件界面
**输出**：组件层级、交互逻辑、可用性建议

### `design` - 设计分析
**核心能力**：格式塔原理、色彩系统、字体层级
**适用**：海报、PPT、设计稿、品牌物料
**输出**：设计原理分析、优缺点评估

### `photo` - 照片分析
**核心能力**：构图法则、用光分析、技术参数推测
**适用**：摄影作品、风景人像、街拍纪实
**输出**：专业摄影评论、技术分析

### `chart` - 图表提取
**核心能力**：图表类型识别、精确数据提取、趋势分析
**适用**：柱状图、折线图、饼图、散点图
**输出**：原始数据、趋势解读

### `reverse` - 提示词逆向
**核心能力**：AI生成特征识别、多模型格式输出
**适用**：AI生成图片、Midjourney/Stable Diffusion作品
**输出**：可复用的提示词，支持多种格式

### `chat` - 聊天提取
**核心能力**：消息类型识别、时间线提取、关键信息摘要
**适用**：微信/QQ聊天截图、对话记录
**输出**：结构化对话内容、时间线、关键信息

### `marketing` - 营销分析
**核心能力**：传播策略、文案评估、转化路径
**适用**：广告海报、宣传物料、营销图片
**输出**：传播效果评估、优化建议

### `default` - 通用分析
**核心能力**：三层分析框架、完整描述
**适用**：无明确专项需求、混合内容、不确定时
**输出**：系统化的内容描述

## 执行要求

- **超时**：调用时**必须设置至少 10 分钟超时**（长图切片 + 多阶段处理耗时较长）
- **工作目录**：必须在**用户当前工作目录**执行，不要 cd 到 skill 目录
- **输出**：分析结果直接输出到终端（stdout），不要保存为文件
- **type选择**：每次调用**必须**根据用户意图选择正确的 --type

---

## 命令格式

```bash
{skill-folder}/img-parse <图片路径> --type <类型> [--prompt "附加要求"]
```

### 参数说明

| 参数 | 必选 | 说明 |
|------|------|------|
| `<图片路径>` | 是 | 本地图片文件路径或 URL |
| `--type <类型>` | **强烈建议** | 解析类型，根据用户意图选择 |
| `--prompt <提示词>` | 否 | 补充分析要求 |
| `--help, -h` | - | 显示帮助 |

---

## 示例

### 正确用法示例

```bash
# 用户说"识别这张截图里的文字" → ocr
{skill-folder}/img-parse ./screenshot.png --type ocr

# 用户说"分析这个APP界面" → ui
{skill-folder}/img-parse ./app.png --type ui

# 用户说"这张海报设计得怎么样" → design
{skill-folder}/img-parse ./poster.png --type design

# 用户说"分析这张照片的构图和用光" → photo
{skill-folder}/img-parse ./photo.jpg --type photo

# 用户说"提取图表里的数据" → chart
{skill-folder}/img-parse ./chart.png --type chart

# 用户说"这个AI图是怎么生成的" → reverse
{skill-folder}/img-parse ./ai-art.png --type reverse

# 用户说"分析这个广告的传播效果" → marketing
{skill-folder}/img-parse ./ad.png --type marketing
```

### 附加要求

```bash
# 在选定类型基础上附加要求
{skill-folder}/img-parse ./doc.jpg --type ocr --prompt "重点关注数字和日期"
{skill-folder}/img-parse ./ui.png --type ui --prompt "评估可访问性"
{skill-folder}/img-parse ./ad.png --type marketing --prompt "目标受众是年轻人"
```

## 注意事项

- 长图会自动切割处理，无需手动干预
- 支持格式：PNG、JPG、JPEG、WebP、BMP、TIFF
- 支持本地路径和 URL（URL 会自动下载到临时目录）
