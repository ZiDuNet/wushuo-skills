# 打包说明 — 潮汐 Skills 发布流程

本文档指导如何将 skill 打包为 zip、更新 skill.md 列表、上传到 OSS。

---

## 1. 打包

### osszip 目录结构

```
osszip/
├── skill.md                # cx-skills-hub，给 AI Agent 用的下载管理 skill
├── 打包.md                  # 本文档（给开发者用）
├── cx-conference.zip       # 会议演讲数据
├── cx-wechat-article.zip   # 微信公众号文章
├── cx-screen.zip           # 潮汐智慧屏管理
├── post-creator.zip        # HTML 海报生成
├── poster-screen.zip       # 海报生成与投屏
├── alt-image-parse.zip     # AI 图片解析
└── alt-tools_web.zip       # 网络搜索与网页读取
```

### 规则

- 只打包目录下有 `SKILL.md` 的 skill 目录
- 目录名即 zip 包名：`cx-conference` → `cx-conference.zip`（不含版本号）
- zip 内保持目录名作为顶层（解压后为 `{skill-name}/SKILL.md`）
- **排除运行时目录**：`.auth/`、`data/`、`node_modules/`、`.git/` 不打入 zip

### 一键打包

扫描所有 `chaoxi_` 开头的父目录，自动找到含 `SKILL.md` 的子目录，排除运行时目录后打包：

```powershell
$base = "D:\AI\chaoxi_skills"
$out  = "$base\osszip"
$exclude = @('.auth', 'data', 'node_modules', '.git')

Get-ChildItem "$base\chaoxi_*" -Directory | ForEach-Object {
    Get-ChildItem $_.FullName -Directory | ForEach-Object {
        $skill = $_
        if (Test-Path "$($skill.FullName)\SKILL.md") {
            $tmp = "$env:TEMP\$($skill.Name)"
            if (Test-Path $tmp) { Remove-Item $tmp -Recurse -Force }
            Copy-Item $skill.FullName $tmp -Recurse -Force
            $exclude | ForEach-Object {
                $p = Join-Path $tmp $_
                if (Test-Path $p) { Remove-Item $p -Recurse -Force }
            }
            Compress-Archive -Path $tmp -DestinationPath "$out\$($skill.Name).zip" -Force
            Remove-Item $tmp -Recurse -Force
            Write-Host "packed: $($skill.Name).zip"
        }
    }
}
```

> **原理**：`Compress-Archive` 不支持排除，所以先复制到临时目录，删掉排除项，再打包，最后清理临时目录。

### 打包单个 skill

```powershell
$skill = "<skill完整路径>"
$name = Split-Path $skill -Leaf
$out = "D:\AI\chaoxi_skills\osszip"
$exclude = @('.auth', 'data', 'node_modules', '.git')

$tmp = "$env:TEMP\$name"
if (Test-Path $tmp) { Remove-Item $tmp -Recurse -Force }
Copy-Item $skill $tmp -Recurse -Force
$exclude | ForEach-Object { $p = Join-Path $tmp $_; if (Test-Path $p) { Remove-Item $p -Recurse -Force } }
Compress-Archive -Path $tmp -DestinationPath "$out\$name.zip" -Force
Remove-Item $tmp -Recurse -Force
Write-Host "packed: $name.zip"
```

## 2. 更新 osszip/skill.md

osszip/skill.md 是 `cx-skills-hub` skill，供 AI Agent 查询可用 skill 列表并下载。

### 何时更新

- 新增 skill 时，添加一行
- 修改 skill 描述时，更新对应行
- 删除 skill 时，移除对应行

### 表格格式

在 `## 可用 Skills 列表` 下的表格中操作，每行格式：

```
| {name} | {一句话描述} | `https://4dbim-chaoxi.oss-cn-shanghai.aliyuncs.com/deploy/skills/{name}.zip` |
```

**name** 取自 skill 目录下 SKILL.md 的 `name` 字段。

**描述** 取自 SKILL.md 的 `description`，精简为一句话。

### OSS 下载路径

- 基础路径：`https://4dbim-chaoxi.oss-cn-shanghai.aliyuncs.com/deploy/skills/`
- 命名：`{目录名}.zip`，不含版本号，始终覆盖为最新版

---

## 3. 版本号同步

每次修改 skill 代码或文档时，以下位置的版本号必须同步更新：

| 位置 | 说明 |
|------|------|
| `chaoxi_*/cx-*/SKILL.md` | 各 skill 自身的 `version` 字段 |
| `README.md` | 目录结构和 skill 卡片中的版本号 |
| `osszip/skill.md` | cx-skills-hub 的 `version` 字段 |

> `cx-` 前缀的 skill 共用同一版本号，统一递增。`alt-` 前缀的 skill 版本独立管理。

---

## 4. OSS 上传

- **Bucket**：`4dbim-chaoxi`
- **前缀**：`deploy/skills/`
- **上传内容**：osszip 下的 `skill.md` + 所有 `.zip`
