# cx-wechat-article — 微信公众号文章撰写

根据主题、素材或已有 Markdown，生成图文并茂的微信公众号文章，输出微信兼容的 HTML，支持多种排版风格。

---

## AI 执行流程

```
用户需求 → 确认风格 → 确定标题 → 生成/优化文章(Markdown) → 用户确认 → 渲染微信 HTML → 输出/推送
```

### 第零步：获取会议数据（如涉及潮汐智能会议）

如果用户想写关于潮汐智能会议演讲的文章，**写文章前必须先获取演讲数据**：

1. 执行 `cx-conference` 的 `auto` 获取会议列表
2. 用户选择会议后，执行 `load-catalog` 加载目录
3. AI 匹配到相关演讲后，执行 `use-presentation` 获取详情
4. `use-presentation` 返回的 `validatedImages` 数组是文章配图的**唯一来源**
5. 拿到数据和图片后，再进入第一步开始写文章

### 第一步：确定排版风格

**规则：用户指定 > 内容自动匹配 > 默认 classic**

#### 自动匹配规则（用户未指定时）

根据文章内容关键词自动选择最合适的风格：

| 风格 | 自动匹配关键词 | 特点 |
|------|---------------|------|
| `classic` | 无明显倾向时默认 | 经典黑白，简洁专业 |
| `tech-blue` | AI、技术、编程、产品、SaaS、数据、互联网、工具 | 科技蓝，现代感 |
| `warm-orange` | 生活、成长、情感、育儿、美食、健身、心理 | 暖橙色调，亲和力 |
| `elegant-green` | 旅行、自然、健康、文化、教育、读书、环保 | 优雅绿，清新自然 |
| `dark` | 深度分析、专栏、长文、评论、哲学 | 深色背景，沉浸感 |

匹配到后**告知用户**选择了哪个风格及原因，用户可以随时要求更换。

#### 手动指定

用户可以直接说"用科技蓝风格"或"换个暖色系"，此时按用户指定执行。

#### 风格定义文件

每个风格的完整配色和 AI 渲染 prompt 定义在 `themes/<风格名>.md` 中，结构如下：

```yaml
name: <风格名>
type: ai
description: "风格描述"
colors:
  background: "#色值"
  text: "#色值"
  primary: "#色值"
  ...
prompt: |
  完整的 AI 渲染指令（配色、结构、每个元素的精确样式）
```

渲染 HTML 时**必须**读取对应主题文件的 `prompt` 部分，将 prompt + 文章 Markdown 内容组合后，由 AI 生成最终的微信 HTML。

### 第二步：确定标题（必做）

**无论什么输入类型，都必须先确定标题再写正文。**

1. 根据文章主题，提供 **3 个不同风格的标题选项**（见下方「标题建议」章节）
2. 暂停，让用户选择或提出修改
3. 用户选定标题后，进入第三步创作正文

> 不要跳过这一步直接写文章。即使内容已明确，也要先让用户确认标题。

### 第三步：判断输入类型并创作

| 用户给的 | AI 应该做的 |
|----------|------------|
| 一个主题/想法 | 从零创作，先给 3 个标题选项，再写正文 |
| 素材/笔记/片段 | 整理扩展为结构化文章 |
| Markdown 文件 | 优化内容 + 直接渲染 |
| 已有文章 + "润色" | 优化表达、去 AI 味、调整结构 |

**创作要求**：
- 文章结构清晰，段落简短（每段 3-5 句）
- 适当使用加粗强调重点
- 在合适位置放置用户提供的图片
- 末尾自然收尾，不要过度总结

**会议数据文章的配图（强制）**：

当文章内容涉及潮汐智能会议演讲时，**必须配图，不可纯文字**。配图来源和流程：

1. **获取图片**：执行 `cx-conference` 的 `use-presentation` 命令后，返回的 `validatedImages` 数组包含所有幻灯片图片的 URL 和可访问状态
2. **筛选图片**：从 `validatedImages` 中选 `accessible: true` 的图片，根据文章段落内容匹配最相关的配图
3. **插入图片**：在对应段落后用 Markdown 图片语法插入：`![图片描述](imageUrl)`
4. **密度要求**：每 2-3 段必须有一处配图，避免纯文字堆砌
5. **图片不可用时换一张**：只使用 `accessible: true` 的图片，如果合适的图不可访问，从同演讲的其他可用图片中挑选替代
6. **文末来源声明**（必须添加，"潮汐智能会议"名称不可改）：

格式参考（可根据文章风格调整排版）：

```
---\n本文内容基于「潮汐智能会议」XXX 会议 · 《演讲标题》 整理\n
```

### 第四步：输出 Markdown 并确认

**先输出 Markdown（可编辑）：**

输出完整的 Markdown 文本，保存为 `.md` 文件。

**然后暂停，询问用户确认：**
- 内容是否满意？需要修改哪些部分？
- 确认无误后，进入第四步渲染 HTML

用户可能：
- 要求修改内容 → 修改后重新确认
- 确认可用 → 进入第四步
- 只要 Markdown，不要 HTML → 流程结束

### 第五步：渲染微信 HTML（用户确认后）

**渲染规则**（详见 `references/wechat-html-spec.md`）：
- 所有样式必须用 **inline style**（微信过滤 `<style>` 和 class）
- 图片 `width: 100%`，正文区最大宽度 `677px`

**HTML 渲染方式**：
1. 读取 `themes/<风格名>.md` 获取该主题的完整 prompt
2. 将 prompt 末尾的 `请转换以下 Markdown 内容：` 后面拼接文章的 Markdown 内容
3. 由 AI 根据这段组合 prompt 生成最终的微信 HTML

### 第六步：推送（可选）

如果用户需要推送到草稿箱，询问微信 AppID 和 AppSecret，然后执行推送脚本（见下方「草稿箱推送」章节）。

## AI 去痕清单

润色文章时，修正以下 AI 痕迹：

| 检测到的 | 换成 |
|---------|------|
| "标志着"、"凸显了" | 具体描述 |
| "不仅…而且…此外" | 自然过渡 |
| "深入探讨"、"全面解析" | 口语化表达 |
| "为了实现这一目标" | 删除 |
| "希望这对您有帮助" | 删除 |
| "让我们一起期待" | 删除或换成个人观点 |

---

## 标题建议

创作文章时，提供 3 个不同风格的标题：

| 类型 | 示例 |
|------|------|
| 悬念式 | "为什么 90% 的人都做错了？" |
| 干货式 | "5 个技巧让你效率翻倍" |
| 观点式 | "别再假装努力了" |

## 草稿箱推送（可选）

如果用户需要直接推送到公众号草稿箱，需要以下配置：

### 前置条件
1. 微信公众号的 AppID 和 AppSecret（微信公众平台 → 设置与开发 → 基本配置）
2. 将当前机器的公网 IP 加入公众号 API 白名单
3. 封面图（用户指定或文章中第一张图）

### 推送命令

```bash
node "{skill-folder}/scripts/push-draft.js" \
  --html "<HTML文件路径>" \
  --title "文章标题" \
  --appid "<AppID>" \
  --secret "<AppSecret>" \
  --cover "<封面图URL或路径>"
```

### 推送流程

1. 用 AppID + AppSecret 获取 access_token
2. 上传封面图为微信永久素材
3. 替换正文中所有外部图片为微信 CDN 地址
4. 创建草稿并推送
5. 返回成功信息

### 常见错误

| 错误码 | 原因 | 解决 |
|--------|------|------|
| `40164` | IP 不在白名单 | 把报错中的 IP 加入公众号后台白名单 |
| `40001` | AppSecret 错误 | 检查 AppSecret 是否正确 |
| `45002` | 内容超限（20k字符/1MB） | 缩短文章或精简 HTML |

> **注意**：推送功能需要 Node.js 18+（内置 fetch），零外部依赖。
