# 潮汐AI赋能方案生成系统

> 输入场景信息 → 自动检索分析 → 动态生成AI赋能宣传材料

## 架构总览

```
课程生成/
├── SKILL.md                              ← 主入口（场景路由 + 通用规则）
├── README.md                             ← 本文件
│
├── references/
│   ├── content-framework.md              ← 四层内容架构 + 措辞规范（所有场景共享）
│   ├── scenario-L1-general.md            ← L1 泛销售场景
│   ├── scenario-L2-industry.md           ← L2 行业/企业场景
│   ├── scenario-L3-enterprise.md         ← L3 企业内部场景
│   ├── scenario-L4-channel.md            ← L4 渠道合作场景
│   └── scenario-L5-school.md             ← L5 学校合作场景
│
├── scripts/
│   ├── feishu-wiki.js                    ← 飞书知识库访问脚本（App凭证认证）
│   └── package.json
│
└── assets/
    └── logo.png                          ← 潮汐AI品牌Logo
```

## 四层内容架构

所有场景生成的材料必须包含四个层次：

| 层次 | 内容 | 目的 |
|------|------|------|
| **业务内容** | 课程设计、课表安排、知识库素材匹配 | 核心价值 |
| **背景内容** | AI行业趋势、最新新闻、生产力数据 | 紧迫感 |
| **形象包装** | 视觉设计、多页排版、专业商务风格 | 专业感 |
| **市场化描述** | CTA（行动号召）、营销用词、价值主张 | 行动力 |

## 五种场景

| 场景 | 触发方式 | 输入 | 输出 | CTA |
|------|----------|------|------|-----|
| **L1 泛销售** | "出个介绍"/"宣传页" | 无 | 竖版HTML/PDF/海报，5-8页 | 联系我们 |
| **L2 行业/企业** | "给工程行业出个方案" | 行业名或公司名 | 竖版HTML/PDF/PPT，8-12页 | 转发给相关人 |
| **L3 企业内部** | 提供会议录音/纪要 | 录音/转写/材料 | 横版HTML/PPT/PDF，10-15页 | 申请预算 |
| **L4 渠道合作** | "渠道合作"/"角色化招生" | 渠道商信息+目标角色 | 竖版HTML/PDF/海报，可贴牌 | 开班报名 |
| **L5 学校合作** | "跟XX学校合作办课" | 学校名称+合作方向 | 视情况 | 期待确认 |

## 执行流程

```
用户输入
  │
  ▼
场景识别 ──→ 判定 L1/L2/L3/L4/L5
  │
  ├── L1 ──→ 直接拉知识库 + AI新闻 → 生成通用介绍
  ├── L2 ──→ WebSearch搜行业 + 知识库 → 生成行业方案
  ├── L3 ──→ 提取录音信息 + WebSearch + 知识库 → 生成深度方案
  ├── L4 ──→ WebSearch搜渠道 + 知识库 → 生成贴牌招生材料
  └── L5 ──→ WebSearch调研学校 + 知识库 → 定向适配方案
  │
  ▼
输出材料（HTML/PDF/PPT/海报）
```

## 知识库访问

所有课程素材来自飞书知识库，通过 `scripts/feishu-wiki.js` 访问：

```bash
# 同步知识库（首次或版本更新时）
node scripts/feishu-wiki.js sync Y6cVwowzViqpEFkLpEvcC0LNnEn 3

# 查看目录
node scripts/feishu-wiki.js --cached tree

# 搜索文档
node scripts/feishu-wiki.js --cached find "关键词"

# 读取文档
node scripts/feishu-wiki.js --cached read-doc <obj_token>
```

缓存策略：基于根节点标题中的版本号自动判断是否需要更新。

## 措辞规范

| 不要说 | 要说 |
|--------|------|
| AI培训 | AI赋能 / 智能化升级 |
| 教 / 教程 | 赋能 / 实战 / 能力跃迁 |
| 课程 | 赋能模块 / 实战单元 |
| 学员 | 参训人员 / 赋能对象 |
| 上课 | 赋能实战 / 密集实操 |

核心：客户看到的是"帮我的团队升级"，不是"又要上课"。

## 课程结构

每个 Unit = 2天 = 4个半天：

| 半天 | 定位 | 产出物 |
|------|------|--------|
| 第1个半天 | AI基础能力建设 | 独立完成基础AI对话任务 |
| 第2个半天 | 业务场景深度实操 | 一个可直接用的业务模板 |
| 第3个半天 | 进阶场景拓展 | 形成个人AI工具箱 |
| 第4个半天 | 综合实战落地 | 一个属于自己的AI提效方案 |

Unit数量 = 赋能天数 / 2（向上取整）。
