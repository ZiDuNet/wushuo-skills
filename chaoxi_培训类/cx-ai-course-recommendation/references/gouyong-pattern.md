# 「够用版」模式：阈值预警 + 趋势分析

> 适用场景：客户明确说「不做顶级算法，够用即可、满足业主验收」。常见于桥梁监测、隧道监测、大坝监测、设备运维等工程监测项目。

## 识别信号

当客户说以下任一句话时，启动「够用版」模式：

- "不用顶级算法"
- "够用版：阈值预警、趋势分析即可满足业主验收"
- "不需要机器学习/深度学习"
- "简单数据分析即可"
- "别太复杂，能交差就行"

## 课表设计原则

| 能力 | 实现方式 | **不用什么** |
|------|---------|-------------|
| ✅ 阈值预警 | 历史数据统计推荐 + 多条件联动规则 | ❌ 不训练 ML 模型 |
| ✅ 趋势分析 | AI 辅助可视化判断 + 数据拐点识别 | ❌ 不上深度学习预测 |
| ✅ 异常过滤 | 规则引擎自动标记 + AI 辅助生成规则 | ❌ 不用复杂信号处理 |
| ✅ 报表生成 | 模板化自动填充 + 自然语言摘要 | ❌ 不改造现有系统 |
| ✅ 数据接入 | 标准化 API/文件读取 + AI 辅助字段映射 | ❌ 不需额外硬件 |

## 课表用词规范

- ✅ "阈值比对判断" "历史数据统计推荐" "规则引擎" "模板填充"
- ❌ "机器学习模型" "神经网络" "深度学习" "训练/调参" "特征工程"

## 三层赋能架构（监测类项目标准结构）

```
决策层（半天）
├─ AI趋势 + 行业智能化路径
└─ 经营者AI工具实操

执行层（2天）
├─ Day1上：AI基础 + 行业提示词
├─ Day1下：报表自动生成 + 数据分析
├─ Day2上：工作流自动化 + 告警规则
└─ Day2下：综合实战 + 部门工具箱

落地层（2天）
├─ Day1上：AI上手 + 平台查询提效
├─ Day1下：异常过滤 + 阈值设置
├─ Day2上：趋势分析 + 报表生成
└─ Day2下：全流程实战 + 个人方案
```

## 课表中必须覆盖的平台功能

根据客户提到的具体系统功能，逐项映射到课表中：

| 客户功能诉求 | 课程映射 | 工作量减少 |
|------------|---------|-----------|
| 后台管理 | 自然语言查询替换手动翻后台 | 查询 5min→10s |
| 数据接入 | API/文件读取 + AI 辅助字段映射 | 数据准备时间减少 |
| 阈值设置 | 喂历史数据→AI 推荐阈值参数 | 调试从多轮试错→一次到位 |
| 预警推送 | 多条件联动规则→自动告警到群 | 告警规则配置提效 |
| 数据存储 | 自动归档（报表生成器中附带） | — |
| 报表自动生成 | 模板填充 + AI 自然语言摘要 | 2h→10min |
| 简单数据分析 | 输入数据→AI 输出趋势 + 对比 | 1h→3min |
| 异常数据过滤 | 一句人话→一条规则，AI 标注 | 清洗时间减 70% |
