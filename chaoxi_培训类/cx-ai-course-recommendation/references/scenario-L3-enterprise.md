# L3 企业内部场景

> 基于会议录音/纪要，给具体企业出深度定制方案。对接人拿去直接给领导汇报。

## 场景定义

| 维度 | 说明 |
|------|------|
| **受众** | 已沟通的企业对接人 + 其上级领导 |
| **目的** | 对接人看完能直接打包成报告提交上级，无需二次加工 |
| **CTA** | 已具备启动条件，建议安排启动会 / 申请预算 |
| **品牌** | 潮汐 AI 品牌完整露出 |

**核心逻辑：** 接收者（对接人）需要说服他的领导。材料要让他觉得"拿这个去汇报，对我有好处"，领导看了觉得"这事值得投钱"。

## 输入要求

**路径 A：会议录音/转写文本**
- 从转写中提取：团队构成、AI 现状、核心痛点、项目信息
- 这是最高价值信息源，提取方法见 SKILL.md「会议转写提取技巧」

**路径 B：用户提供的材料**（Word/飞书文档等）
- 读取文档内容，提取关键信息

**路径 C：都没有**
- 收集保底信息：
  1. **公司名称**（必须）
  2. **需要赋能的人员类型/岗位**（必须）
  3. **行业**（如有就问）
  4. **团队规模**（可选）
  5. **已知痛点或需求**（可选）

> 保底信息 = 公司名称 + 人员类型，没有就不能往下走

## 执行流程

1. **信息提取**（从录音/材料/用户输入）
2. **客户画像分析**
   - 团队构成：每个角色是谁、做什么、AI 工具使用现状
   - AI 成熟度分级：零基础 / 工具混乱期 / Agent 入门期 / 重度用户
   - 核心痛点：从材料中提取反复出现的问题（原文依据，不猜测）
   - 决策者 vs 使用者的诉求差异
   - WebSearch 搜索公司全称：行业、主营业务、规模、最新动态、AI/数字化转型现状
3. **询问赋能天数**（必须先问！提供 2/4/6 天等选项）
4. **拉取知识库素材**
   - 先读「课程体系总览」→ 获取课程信息、技能编号、学习路径
   - 浏览完整目录树
   - 根据客户画像自主决定读取哪些文档
   - WebSearch 搜索该行业 AI 应用案例
5. **课程匹配与定制**
   - 根据行业 + 角色 + AI 成熟度匹配知识库素材
   - 根据真实痛点定制业务场景
   - 每个半天的详细内容规划
6. **生成输出材料**

## 课程匹配详细规则

**素材匹配：**
1. 从知识库获取所有可用课程和技能模块
2. 根据客户行业 + 团队角色 + AI 成熟度选择最匹配的素材
3. 每个半天必须有对应的知识库文档支撑——没有对应素材的模块标注"需定制"
4. 一个半天可以组合多个知识库文档

**内容定制维度：**

| 维度 | 怎么做 | 示例 |
|------|--------|------|
| AI 成熟度 | 调整第 1 个半天的深度 | 零基础→多讲基础；有经验→快速过，重点场景 |
| 行业特性 | 用客户行业的术语、案例、数据 | 金融→风控报告；制造→生产数据分析 |
| 团队角色 | 不同角色安排不同业务场景 | 办公岗→PPT/Excel；技术岗→代码/自动化 |
| 真实痛点 | 从材料提取痛点作为场景 | "每周写 3 份 PPT"→PPT 提效场景 |

**每个半天的输出规范：**
- **主题**：一行标题，说清楚这个半天解决什么问题
- **时段安排**：具体到每节课的时间、类型（理论/实操/讨论）
- **教学内容**：从知识库素材提炼的核心知识点，用业务语言描述
- **实战任务**：学员做什么练习，用客户真实场景
- **预期产出**：半天结束后学员能拿到什么成果

## 输出格式

| 格式 | 适用场景 | 说明 |
|------|----------|------|
| **HTML**（默认） | 投屏/打印/转发 | 横版 PPT 风格，10-15 页，适配打印 |
| PPT/PPTX | 内部正式汇报 | 可直接用于汇报会议 |
| PDF | 打印装订 | 正式文件 |

> L3 默认横版（PPT 风格），因为对接人要打印或投屏给领导看。

## 设计要点

- **横版 PPT 风格**：适合打印和投屏，对接人直接拿去汇报
- **10-15 页**：信息密度高，但要清晰
- **数据说话**：每个结论有数字支撑
- **客户真实场景**：痛点、对比、案例全部来自客户材料
- **半天课表完整展开**：时段 + 内容 + 实战任务 + 产出
- **形象要够"重"**：类比卖华为手机——一米长的详情页比一张海报有说服力
